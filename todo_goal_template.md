## 1. 清晰的最终状态 (Clear Final State)

构建一个基于 Godot 4.6 的桌面桌宠（Desktop Mascot）应用，具体包含以下核心系统：

1. **透明无边框窗口系统**：
   * 继承并优化现有的 `transparent_bg` 与 `borderless` 窗口属性。
   * 支持通过特定按键/鼠标中键拖拽窗口位置。
2. **绘图与自定义导入编辑器 (Mascot Editor UI)**：
   * **画板模块**：应用内包含简易画板，允许用户用笔刷手动绘制“身体”、“左手”、“右手”、“尾巴”和“头部”等贴图。
   * **本地导入模块**：提供图片导入按钮，允许用户选择本地图片（`.png`/`.jpg`），并将其裁剪/映射到对应的肢体节点上。
3. **2D 骨骼与肢体绑定系统**：
   * 在 Godot 中构建可复用的 2D 骨骼结构（或使用层级式 Parent-Child 旋转关节节点）。
   * 无论是手绘还是导入的图片，都会自动绑定到骨骼模型对应的插槽上，确保动作时关节过渡自然。
4. **双风格动作引擎 (Cute & Bizarre Styles)**：
   * **可爱风格**：动作圆润平缓，具有缓入缓出（Ease In/Out）效果。
   * **鬼畜（抽象）风格**：动作高频、摇摆幅度极大、伴随骨骼轻微错位或拉伸，充满趣味性与视觉冲击。
5. **全局/局部键鼠交互轮询机制**：
   * 监听系统的鼠标点击与键盘按键。
   * **左右挥爪轮询**：第 1 次按键/点击触发“挥动左手”，第 2 次触发“挥动右手”，交替循环。
   * **同步晃头**：挥爪时伴随头部的左右晃动物理摆反馈。

## 2. 明确的验收标准 (Explicit Acceptance Criteria)

- [ ] **画板与导入测试**：
  * 用户可以在画板上手绘，并且手绘内容能即时应用到桌宠模型上。
  * 导入本地图片时，能正确裁剪并加载到对应的骨骼节点（不发生严重拉伸或位置偏移）。
- [ ] **骨骼动画完整性**：
  * 关节旋转（如挥爪、摇尾、晃头）时，贴图关节处紧密相连，无明显脱节或穿模。
- [ ] **键鼠交互与轮询**：
  * 每次键盘输入或鼠标点击，爪子动作必定在“左爪挥动”与“右爪挥动”之间交替切换（L -> R -> L -> R ...）。
  * 挥爪时，头部有明显的摆动/回弹动画，且动画速度能根据按键频率（Excitement 兴奋度）动态加速。
- [ ] **风格切换效果**：
  * 界面上存在“可爱”与“鬼畜”两个风格切换按钮。
  * 切换到“鬼畜”风格时，动作速度提高一倍以上，晃动幅度和旋转角度显著增大，且可能伴随轻微的随机肢体位移。
- [ ] **无报错运行**：
  * 运行游戏时控制台无任何 `GDScript` 运行期报错（Errors）或崩溃。
